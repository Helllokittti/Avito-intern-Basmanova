# API Тесты для Avito Микросервиса
# Avito intern Basmanova

**Базовый URL:** `https://qa-internship.avito.com`

**Поддерживаемые версии API:**
- Version 1: `/api/1/*` - создание, получение, статистика
- Version 2: `/api/2/*` - удаление, статистика

## Структура тестов

tests
- test_api_delete.py # DELETE /api/2/item/{id} (Тест 4.1)
- test_api_E2E.py # E2E сценарий (Тест 6.1)
- test_api_id.py # GET /api/1/item/{id} (Тест 2.1)
- test_api_id_no_name.py # POST без name (Тест 1.2, 1.4)
- test_api_price.py # POST с отрицательной ценой (Тест 1.3)
- test_GET_items_by_sellerId.py # GET /api/1/{sellerId}/item 
- test_api_statistic_1.py # GET /api/1/statistic/{id} (Тесты 3.1-3.3)
- test_api_statistic_2.py # GET /api/2/statistic/{id} (Тесты 5.1-5.2)


#### test_api_id_no_name.py (Тесты 1.2 и 1.4)
- **Тест 1.2:** Создание объявления без обязательного поля `sellerId`
  - Ожидаемый результат: HTTP 400, сообщение об ошибке
  - Фактический результат: HTTP статус "400" (строка)
  
- **Тест 1.4:** Создание объявления с пустым именем (`name = ""`)
  - Ожидаемый результат: HTTP 400, сообщение об ошибке
  - Фактический результат: HTTP статус "400" (строка), сообщение "поле name обязательно"

#### test_api_price.py (Тест 1.3)
- **Тест 1.3:** Создание объявления с отрицательной ценой (`price = -100`)
  - Ожидаемый результат: HTTP 400, ошибка валидации
  - Фактический результат: HTTP 200, объявление успешно создается
  - **Вывод:** Отсутствует проверка на отрицательное значение цены

#### test_api_id.py (Тест 2.1)
- **Тест 2.1:** Получение объявлений по `sellerId` через GET /api/1/{sellerId}/item
  - Проверяет, что возвращаются объявления с корректными значениями полей
  - Фактический результат: name = "", price < 0, статистика может быть некорректна
  - **Вывод:** Отсутствует серверная валидация и контроль целостности данных

#### test_GET_items_by_sellerId.py (Тест 2.1 расширенный)
- Расширенная проверка получения объявлений по sellerId
- Валидация каждого поля возвращаемых объявлений
- Проверка бизнес-правил для name, price, статистики

#### test_api_statistic_1.py (Тесты 3.1, 3.2, 3.3)
- **Тест 3.1:** Успешный запрос статистики для существующего id
  - Ожидаемый результат: HTTP 200, структура {contacts, likes, viewCount}
  - Фактический результат: HTTP статус "200" (строка), статистика получена
  
- **Тест 3.2:** Запрос статистики для несуществующего id
  - Ожидаемый результат: HTTP 400, сообщение об ошибке
  - Фактический результат: Совпадает с ожидаемым
  
- **Тест 3.3:** Серверная ошибка при невалидном вводе
  - Ожидаемый результат: HTTP 500
  - Фактический результат: HTTP 500, сообщение "Status 500: Серверная ошибка"

#### test_api_delete.py (Тест 4.1)
- **Тест 4.1:** E2E сценарий удаления (create → delete → verify)
  1. Создание объявления через POST /api/1/item
  2. Удаление через DELETE /api/2/item/{item_id}
  3. Проверка GET /api/1/item/{item_id} → 404 Not Found
  - **Результат:** Удаление проходит успешно, данные после удаления недоступны

#### test_api_statistic_2.py (Тесты 5.1, 5.2)
- **Тест 5.1:** Успешный запрос статистики v2 для существующего id
  - Ожидаемый результат: HTTP 200, структура {contacts, likes, viewCount}
  - Фактический результат: HTTP статус "200" (строка), статистика получена
  
- **Тест 5.2:** Запрос статистики v2 для несуществующего id
  - Ожидаемый результат: HTTP 404
  - Фактический результат: "Status 404: OK", данные не найдены

#### test_api_E2E.py (Тест 6.1)
- **Тест 6.1:** E2E сценарий (создание и получение объявления)
  1. POST /api/1/item с валидными данными
  2. Получение id созданного объявления
  3. GET /api/1/item/{id}
  - **Результат:** Объявление успешно создается, данные совпадают

## Требования

- **Python**: 3.9 или выше
- **pip**: последняя версия
- **Операционная система**: Windows / macOS / Linux
- **Доступ к интернету**: для отправки запросов к API
- **Библиотеки**: pytest, requests (устанавливаются автоматически)

## Запуск всех тестов

pytest tests/ -v

## Создание файла со всеми тестами(не обязательно)

pytest tests/ --html=report.html
